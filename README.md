# PTCG AI Battle Challenge — Complete Solution

## Overview

This repository contains a competitive AI agent for The Pokémon Company's
PTCG AI Battle Challenge (Simulation Category), built on the HEROZ/UTokyo
"cabt" engine via Kaggle Environments.

**Core Approach:** Information-Set Monte Carlo Tree Search (ISMCTS) with:
- Opponent belief modeling (determinized hidden-information sampling)
- Engine-native forward simulation via `search_begin`/`search_step`
- Data-driven heuristic evaluation (6-axis board scoring)
- Anytime algorithm with strict time-budget enforcement

## 1. Running a Local Match

```python
from kaggle_environments import make

# Read your deck
with open("deck.csv") as f:
    deck = [line.strip() for line in f if line.strip()]

# Create environment
env = make("cabt", configuration={"decks": [deck, deck]})

# Run a match (agent vs agent, or swap in opponents)
env.run(["main.py", "main.py"])

# Built-in render
env.render(mode="html")
```

For direct scripting (without kaggle_environments):
```python
from game import battle_start, battle_select, battle_finish, visualize_data

state = battle_start(deck0, deck1)
# ... call battle_select with your agent's choices ...
battle_finish()
print(visualize_data())  # plain-text board dump
```

## 5. Agent Architecture

### Belief Model (`BeliefModel` class in main.py)
Tracks observed opponent cards from logs/discards and samples plausible
determinizations for ISMCTS. Uses the engine's internal randomization for
hidden card assignment when explicit sampling isn't available.

### ISMCTS Engine (`ISMCTSEngine` class in main.py)
- Samples up to `MAX_DETERMINIZATIONS` (8) opponent states per turn
- For each, uses `search_begin()` to create a hypothetical rollout
- Evaluates each top-level action via `search_step()` + random rollout
- UCB1 bandit selection for action exploration within simulations
- Returns action with highest average expected value

### Heuristic Evaluation (`evaluate_state()` in main.py)
Six-axis board scoring, all computed from `CardData`/`Attack` metadata:
1. **HP differential** — total remaining HP (active + bench)
2. **Prize differential** — prizes taken vs given
3. **Energy efficiency** — total energy attached across board
4. **Type matchup** — weakness/resistance from CardData fields
5. **Bench development** — evolved forms + energy on bench
6. **Retreat safety** — active's retreat cost (lower = better)

### SelectType Handling
All select types are handled:
- `MAIN` → Full ISMCTS search
- `YES_NO`, `COUNT`, `SPECIAL_CONDITION` → Context-aware heuristic
- `ENERGY`, `EVOLVE`, `ATTACK`, `CARD`, `ATTACHED_CARD`,
  `CARD_OR_ATTACHED_CARD` → Heuristic with context-specific logic
- `SWITCH`/`RETREAT` → Best bench Pokémon by HP + type matchup

## 6. Configuration

Key parameters in `main.py` (top section):

| Parameter | Default | Description |
|-----------|---------|-------------|
| `TIME_BUDGET_PER_TURN` | 2.5s | Max time per turn (leave margin) |
| `ISMCTS_SIMULATIONS` | 200 | Max simulations per determinization |
| `MAX_DETERMINIZATIONS` | 8 | Opponent state samples per turn |
| `UCB_C` | 1.414 | UCB1 exploration constant |

Adjust `TIME_BUDGET_PER_TURN` based on the competition's per-turn limit.
The algorithm is **anytime** — it always returns the best move found
within the budget.
